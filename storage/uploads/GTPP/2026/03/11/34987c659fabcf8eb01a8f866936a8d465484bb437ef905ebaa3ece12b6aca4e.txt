<h1>Módulo Active</h1>

<p>
  O <strong>Módulo Active</strong> é responsável pela gestão de ativos dentro do sistema,
  concentrando componentes de interface, regras de montagem de dados, mapeamentos e contratos de tipagem.
</p>

<h2>🧠 Descrição Arquitetural</h2>

<h3>🔹 Active.tsx</h3>
<p>
  Arquivo principal do módulo, responsável por orquestrar os componentes, controlar o fluxo da tela
  e integrar as funcionalidades de listagem, formulário e configuração.
</p>

<h3>🔹 Adapters/Adapters.tsx</h3>
<p>Camada de adaptação entre os dados vindos da API e o formato utilizado pela aplicação.</p>
<ul>
  <li>Normalizar dados externos</li>
  <li>Converter estruturas de resposta</li>
  <li>Preparar dados para uso nos componentes internos</li>
</ul>
<h3>🔹 Interfaces/Interfaces.ts</h3>
<p>Define os contratos de tipagem do módulo.</p>
<ul>
  <li>Centralizar interfaces e tipos TypeScript</li>
  <li>Garantir padronização dos modelos de dados</li>
  <li>Aumentar a segurança e previsibilidade do código</li>
</ul>

<h3>🔹 Component/BuildFunction/BuildFunction.ts</h3>
<p>Responsável por funções de construção e preparação de estruturas de dados.</p>

<ul>
  <li>Montar payloads para envio</li>
  <li>Criar objetos de configuração</li>
  <li>Preparar dados para consumo em outros componentes</li>
</ul>

<h3>🔹 Component/DataMapper/DataMapper.ts</h3>
<p>Camada de mapeamento de dados entre:</p>
<ul>
  <li>Estrutura da API</li>
  <li>Estrutura interna da aplicação</li>
  <li>Estrutura esperada pelos componentes de UI</li>
</ul>
<p>Ajuda a manter o desacoplamento entre backend e frontend.</p>

<h3>🔹 Component/FormActive/FormActive.tsx</h3>
<p>Componente responsável pela tela de cadastro/edição do ativo.</p>
<ul>
  <li>Estado do formulário</li>
  <li>Validações</li>
  <li>Submissão de dados</li>
  <li>Integração com schemas e mapeadores</li>
</ul>

<h4>FormSchema</h4>
<p>Contém os schemas de validação e definição estrutural dos formulários:</p>
<ul>
  <li>FormActive.schema.ts → Regras do formulário principal do ativo</li>
  <li>FormAddress.schema.ts → Regras do formulário de endereço</li>
  <li>FormVehicle.schema.ts → Regras do formulário de veículo</li>
</ul>

<p>Essa separação melhora:</p>
<ul>
  <li>Organização</li>
  <li>Reuso</li>
  <li>Manutenibilidade</li>
  <li>Clareza das regras de negócio</li>
</ul>

<h3>🔹 Component/ListAddItem/ListAdd.tsx</h3>
<p>
  Componente responsável por gerenciar listas dinâmicas de itens, permitindo inclusão e manipulação
  de registros associados ao ativo.
</p>

<h3>🔹 Component/ActiveTable.tsx</h3>
<p>Responsável pela exibição tabular dos ativos.</p>
<ul>
  <li>Renderização da listagem</li>
  <li>Exibição de colunas e ações</li>
  <li>Integração com configurações de tabela</li>
</ul>

<h3>🔹 Component/ConfigurationTable/ConfigurationTable.ts</h3>
<p>Define a configuração estrutural da tabela, como:</p>
<ul>
  <li>Colunas</li>
  <li>Formatação de dados</li>
  <li>Regras de exibição</li>
  <li>Comportamento visual</li>
</ul>

<h2>🏗️ Visão Arquitetural Resumida</h2>

<ul>
  <li><strong>Active.tsx</strong> → Orquestrador do módulo</li>
  <li><strong>Adapters / DataMapper / BuildFunction</strong> → Camada de transformação e preparação de dados</li>
  <li><strong>Interfaces</strong> → Contratos e tipagem</li>
  <li><strong>FormActive + Schemas</strong> → Entrada e validação de dados</li>
  <li><strong>ActiveTable + ConfigurationTable</strong> → Exibição e configuração de dados</li>
  <li><strong>ListAddItem</strong> → Gestão de listas dinâmicas</li>
</ul>

<h1>📘 Contratos e Modelos de Dados (Interfaces)</h1>

<p>
  Esta seção documenta os principais <strong>contratos de dados</strong> do Módulo Active.
  Essas interfaces representam o <strong>modelo de domínio</strong>, os <strong>dados de formulário</strong>,
  os <strong>dados vindos da API</strong> e as <strong>estruturas usadas pela UI</strong>.
</p>

<p>Elas ficam centralizadas em <code>Interfaces/Interfaces.ts</code> e servem como:</p>

<ul>
  <li>Contrato entre frontend e backend</li>
  <li>Fonte de verdade para tipagem</li>
  <li>Base para validações e mapeamentos</li>
  <li>Documentação viva do domínio de Ativos</li>
</ul>

<h2>🧩 Conjunto de Dados da Tela</h2>

<h3>ActiveTableData</h3>
<p>Representa o pacote completo de dados que a tela de Ativos precisa para funcionar.</p>

<p>Inclui:</p>
<ul>
  <li>Dados do ativo</li>
  <li>Dados do veículo (se aplicável)</li>
  <li>Listas de apoio (motoristas, empresas, unidades, tipos, etc)</li>
</ul>

<pre><code>
export interface ActiveTableData {
  active: ActiveFormValues;
  vehicle: VehicleFormValues;
  driver: Driver[];
  company: Company[];
  unit: Unit[];
  activeType: ActiveType[];
  fuelType: FuelType[];
  departament: Departament[];
}
</code></pre>

<h2>🧩 Props do Formulário</h2>

<h3>FormActiveProps</h3>
<p>Define os dados que o formulário de Ativo pode receber, permitindo:</p>

<ul>
  <li>Abrir em modo edição</li>
  <li>Controlar abertura/fechamento de modal</li>
  <li>Injetar dados vindos da API</li>
</ul>

<pre><code>
export interface FormActiveProps {
  apiData?: {
    active?: ActiveFormValues;
    departament?: Departament[];
    vehicle?: VehicleFormValues;
    driver?: Driver[];
    company?: Company[];
    unit?: Unit[];
    activeType?: ActiveType[];
    fuelType?: FuelType[];
  };
  openModal?: Dispatch&lt;SetStateAction&lt;boolean&gt;&gt;;
}
</code></pre>
